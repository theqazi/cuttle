## New surface introduced by v1.1 edits

Each v1.1 edit was reviewed for whether it introduces new threat surface that v1 did not have.

| Edit                                               | New surface?                                                                                                                                                                                                                                       |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| §6.1.1 helper-hash field in `CredentialRecord`     | Adds a hash-storage requirement; no new attack surface beyond the existing `CredentialRecord` storage trust boundary.                                                                                                                              |
| §6.1.1 in-memory zeroization                       | Constrains OQ-1; does not introduce new surface. Note: TypeScript implementations would need native binding modules, which themselves are a small supply-chain surface. Flagged for TDD §2.                                                        |
| §6.1.5 attestation-provenance-separation primitive | NEW PRIMITIVE in policy gate. Requires TDD §3 to specify TTY-input-event capture, persistence across the input-to-gate path, and tamper-resistance of the provenance tag. If the provenance tag itself can be forged, T-001 reopens.               |
| §6.1.5 cross-session memory quarantine             | NEW SUBSURFACE. Quarantine area is itself a storage location whose ACL matters; if the quarantine area is world-readable, an unrelated attacker can read pending-promotion content. Flagged for TDD §6 + privacy review per D-09.                  |
| §6.1.2 row L5 operator review queue                | NEW SUBSURFACE. Review queue is itself state that the model should not be able to manipulate. If the model can write to the review queue's storage location through any other tool, the queue's integrity is compromised. Flagged for TDD §3.      |
| §6.1.5 escape-hatch table operator-runtime-loaded  | Reduces attack surface (table no longer model-readable); ships empty by default reduces it further. Net: closes more than it opens.                                                                                                                |
| §8 case 6 lockfile in `~/.cuttle/run/`             | NEW SUBSURFACE. Lockfile location must itself be tamper-resistant. If the outer Cuttle process can be killed without removing the lockfile, the lockfile becomes a deadlock surface (denial-of-service against future starts). Flagged for TDD §3. |

**Net assessment:** v1.1 introduces 4 new sub-surfaces (TTY-provenance primitive, memory quarantine area, L5 review queue storage, nested-harness lockfile). Each has been flagged for TDD-grain treatment. None reaches PRD-grain CRITICAL. The closure of 4 v1 CRITICALs + 3 cross-cuts is net-positive against the surface added.
